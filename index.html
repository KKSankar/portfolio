HTML5 Semantics Project

HTML5 Semantic Elements. Semantic HTML elements clearly describe it's meaning in a human and machine readable way. Elements such as header , footer and article are all considered semantic because they accurately describe the purpose of the element and the type of content that is inside them. Few benefits include: Search engines will consider its contents as important keywords to influence the page's search rankings. Screen readers can use it as a signpost to help visually impaired users navigate a page. By defining block-level elements as semantic elements, we tell the machines what they are, so the machines can render elements appropriately.On a screen, the browser will display text wrapped in an em in italics. But on an audio device for the visually impaired, the text will be pronounced with a corresponding emphasis, just like a friend would say it.
